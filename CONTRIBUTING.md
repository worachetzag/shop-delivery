# Contributing to Samsung Panich Delivery System

Thank you for your interest in contributing to the Samsung Panich Delivery System! This document provides guidelines and information for contributors.

## 🚀 Getting Started

### Prerequisites
- Python 3.9+
- Node.js 16+
- Git
- LINE Developer Account (for LIFF integration)

### Development Setup
1. Fork the repository
2. Clone your fork
3. Set up the development environment
4. Create a feature branch
5. Make your changes
6. Test your changes
7. Submit a pull request

## 📋 Development Guidelines

### Code Style
- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add comments for complex logic
- Write docstrings for functions and classes

### Commit Messages
Use clear, descriptive commit messages:
```
feat: add payment status check API
fix: resolve cart update issue
docs: update API documentation
test: add unit tests for order tracking
```

### Pull Request Process
1. Ensure your code follows the project's style guidelines
2. Add tests for new functionality
3. Update documentation if needed
4. Ensure all tests pass
5. Request review from maintainers

## 🧪 Testing

### Backend Testing
```bash
cd shop_delivery
python manage.py test
```

### Frontend Testing
```bash
cd frontend
npm test
```

## 📚 Documentation

### API Documentation
- Update `API_Endpoints_Summary.md` for new endpoints
- Add docstrings to views and serializers
- Update `SCOPE.md` for scope changes

### Code Documentation
- Add inline comments for complex logic
- Write comprehensive docstrings
- Update README.md for new features

## 🔒 Security

### Security Guidelines
- Never commit sensitive information (API keys, passwords)
- Use environment variables for configuration
- Follow PDPA compliance guidelines
- Implement proper authentication and authorization

### Reporting Security Issues
If you discover a security vulnerability, please:
1. Do not create a public issue
2. Contact the development team directly
3. Provide detailed information about the vulnerability
