โฟลว์การใช้งานผ่าน LINE

เอกสารนี้อธิบายภาพรวมให้ลูกค้าเข้าใจว่าระหว่าง LINE กับหน้าร้านออนไลน์เชื่อมกันอย่างไร — เปิดจากมือถือหรือคอมพิวเตอร์ได้ แชร์ลิงก์ด้านล่างให้ผู้อื่นดูได้

เข้าร้านค้า · ลิงก์หน้านี้:

1) เปิดจาก LINE (เมนู / ปุ่มในแชท / แจ้งเตือน)

flowchart TD
  A(["ผู้ใช้แตะลิงก์ใน LINE"]) --> B{"ชนิดลิงก์"}
  B -->|"liff.line.me"| C["LINE พาไปที่หน้าร้านตามที่ตั้งไว้"]
  B -->|"ลิงก์เว็บตรง"| D["เปิดหน้าร้านในเบราว์เซอร์"]
  C --> E["โหลดหน้าร้านออนไลน์"]
  D --> E
  E --> F{"มีพารามิเตอร์พิเศษจาก LINE"}
  F -->|"มี เช่น liff.state"| G["ระบบส่งไปหน้าที่ถูกต้อง"]
  F -->|"ไม่มี"| H["แสดงหน้าที่เปิดมา"]
  G --> I(["เห็นหน้าสินค้า / ตะกร้า / ออเดอร์ตามที่เลือก"])
  H --> I

2) กรณีที่อยู่ขึ้นคำว่า liff.state

LINE บางครั้งซ่อนชื่อหน้าที่ต้องการไว้ในพารามิเตอร์นี้ — หลังโหลดเสร็จระบบจะพาไป URL ปกติ เช่น /customer/orders/7 

flowchart LR
  subgraph LINE["ใน LINE"]
    U(["ความหมายของหน้า เช่นรายละเอียดออเดอร์"])
  end
  subgraph Web["ในเบราว์เซอร์คำแรก"]
    URL1["ที่อยู่อาจมี ?liff.state=..."]
  end
  subgraph Fixed["หลังประมวลผล"]
    Canon["ที่อยู่ชัด เช่น /customer/orders/7"]
  end
  U --> URL1 --> Canon

3) แจ้งเตือนสถานะคำสั่งซื้อ

flowchart TD
  S(["ร้านหรือระบบอัปเดตสถานะออเดอร์"]) --> N(["ส่งข้อความแจ้งใน LINE"])
  N --> F(["มีปุ่มหรือลิงก์ดูรายละเอียด"])
  F --> U(["ผู้ใช้แตะ"])
  U --> P(["เปิดหน้ารายละเอียดออเดอร์ที่ตรงกับครั้งที่แจ้ง"])

4) เข้าสู่ระบบด้วย LINE (ภาพรวม)

sequenceDiagram
  participant C as ลูกค้า
  participant L as แอป LINE
  participant S as หน้าร้านออนไลน์
  participant B as ระบบร้าน

  C->>L: เปิดหรือแตะล็อกอิน
  L->>S: เปิดหน้าร้าน
  S->>B: เริ่มเข้าสู่ระบบผ่าน LINE
  B->>L: ขอยืนยันตัวตน
  L->>B: ยืนยันแล้ว
  B->>S: ส่งกลับพร้อมสถานะเข้าระบบ
  S->>C: แสดงหน้าร้านหลังล็อกอิน
ซำสูงพาณิชย์ — เอกสารอธิบายการทำงานเบื้องต้น ไม่ใช่ข้อผูกพันทางสัญญา
